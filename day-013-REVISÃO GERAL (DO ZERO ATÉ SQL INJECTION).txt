💻 PROGRAMAÇÃO EM C (BASE PARA ENTENDER EXPLOITS)

Você não virou programador — e nem precisava.
Você aprendeu o suficiente para entender falhas, que é o objetivo real.

Palavras-chave importantes

char → tipo de dado para texto

buffer → espaço na memória para guardar dados

strcpy() → copia texto SEM verificar tamanho (perigoso)

string.h → biblioteca de manipulação de texto

buffer overflow → quando dados ultrapassam o limite do buffer

📌 Você entendeu algo crucial:

O problema não é o atacante, é o código mal escrito.

💥 BUFFER OVERFLOW (O QUE VOCÊ REALMENTE APRENDEU)

Você aprendeu que:

Programas podem ser explorados se não validarem entradas

Isso pode causar:

Crash

Execução de código

Acesso indevido

Evidência prática

Quando apareceu:
